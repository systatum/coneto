# Vite & Webpack Setup

Coneto works out of the box in both Vite and Webpack projects.

## TypeScript Setup (Required for both)

If TypeScript throws an error on CSS imports like:

```
Cannot find module '@systatum/coneto/shared.css' or its corresponding type declarations.
```

Create a `global.d.ts` file in your project root (or `src/`) with:

```ts
declare module "*.css";
```

## Vite

In some edge cases, such as `RichEditor`, or `DocumentViewer` that uses worker-based processing, you may need to adjust Vite configuration.

This is only required if you encounter dependency or worker-related issues.

```ts
import { defineConfig } from "vite";

export default defineConfig({
  optimizeDeps: {
    exclude: ["@systatum/coneto"],
  },
});
```

## Webpack

No extra configuration is required.

Most setups (Next.js, CRA, or custom Webpack configs with style-loader and css-loader) already support Coneto automatically.
