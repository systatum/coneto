### Extending Theme 🎨

You can extend specific styles without rewriting the whole theme configuration.

This is useful when you only want to customize a small part of an existing theme, such as changing button colors, text colors, borders, or backgrounds, while still keeping the default styles from the original theme.

```ts
import {
  createBodyTheme,
  createButtonTheme,
  createTheme,
  darkButton,
} from "@systatum/coneto/theme";

const bodyTheme = {
  backgroundColor: "#fdebce",
  textColor: "#1a1008",
  borderColor: "#6b4c2a",
};

createTheme("parchment", "dark", {
  body: createBodyTheme(bodyTheme),
  button: createButtonTheme(
    bodyTheme,
    darkButton,
    {
      default: {
        backgroundColor: "red",
      },
    },
    {
      default: {
        backgroundColor: "pink",
        textColor: "blue",
      },
    }
  ),
});
```

In this example:

- `bodyTheme` defines the shared colors used by the theme
- `createBodyTheme` creates the body styles from the shared theme values
- `createButtonTheme` creates the button styles using the same theme values
- The third argument customizes the background color of the default button
- The fourth argument pretty much overwrites the third argument, by customizing the default button's background and text color

This lets you customize only the parts you need while keeping the default theme styles.

### Using the Theme

After creating a `theme`, you can use it with the `Theme` provider component as follow:

```tsx
import { Theme, ThemeMode } from "@systatum/coneto/theme";

function App() {
  const [mode, setMode] = useState<ThemeMode | "parchment">("parchment");

  return (
    <Theme mode={mode}>
      <Content setMode={setMode} />
    </Theme>
  );
}
```

Themes created with `createTheme` are automatically registered globally, so you do not need to pass `themes` again into the provider.
